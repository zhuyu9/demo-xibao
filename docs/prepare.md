# 📋 【研发需求单】“镇水犀宝” AI 交互 Python Demo 开发指南

**研发背景**：我们需要用 Python + Gradio（或同类框架）快速搭建一个本地网页 Demo，用于验证“四川话智能毛绒玩具”的对话延迟和人设体验。

## 1. 核心业务工作流 (Pipeline)
前端网页按住按钮说话 -> 本地音频录制 -> **[实时 ASR]** 转文字 -> **[LLM]** 处理回复 -> **[实时 TTS]** 合成四川话音频 -> 前端自动播放。

## 2. 调用的阿里云百炼模型选型清单
*(请务必使用以下最新模型，确保极低延迟)*

*   **🧠 大语言模型 (LLM)**
    *   **模型版本**：`qwen3.5-plus`
    *   **API 文档**：[qwen3.5-plus API 接口说明](https://bailian.console.aliyun.com/cn-beijing/?spm=5176.29619931.J_AHgvE-XDhTWrtotIBlDQQ.13.3dce521c1Kd2Hi&tab=api#/api/?type=model&url=3016808)
    *   **开发要求**：需配置我们在业务端设定好的 `System Prompt`（系统提示词），并开启 `stream=True` 流式输出。

*   **👂 语音识别 (ASR)**
    *   **模型版本**：推荐优先使用 `qwen3-asr-flash-realtime`（或 `Paraformer-realtime`）
    *   **API 文档**：[qwen3-asr-flash-realtime 文档](https://bailian.console.aliyun.com/cn-beijing/?spm=5176.29619931.J_AHgvE-XDhTWrtotIBlDQQ.13.3dce521c1Kd2Hi&tab=doc#/doc/?type=model&url=2989727)
    *   **开发要求**：要求能够做到“边说边识别”，用户一松开按钮，文字立马就绪。

*   **🗣️ 语音合成 (TTS) —— 核心灵魂**
    *   **模型版本**：`qwen3-tts-instruct-flash-realtime` 或 `CosyVoice`
    *   **音色指定**：**Eric 四川话**（这个非常重要，是产品人设的核心！）
    *   **API 文档**：[qwen3-tts-instruct 文档](https://bailian.console.aliyun.com/cn-beijing/?spm=5176.29619931.J_AHgvE-XDhTWrtotIBlDQQ.13.3dce521c1Kd2Hi&tab=doc#/doc/?type=model&url=2938790)
    *   **开发要求**：接收到 LLM 的流式文本后，立刻边合成边播放（Streaming 模式），决不能等整段话生成完再播。

## 3. Demo 界面与交互要求 (建议用 Gradio 实现)
1.  **界面极简**：页面上需要一个大大的**“按住说话 / 录音”**按钮。
2.  **可视化调试**：页面下方请分两个文本框，分别实时显示：
    *   `用户说的话 (ASR识别结果)`
    *   `犀宝的回复 (LLM生成的文字)`
3.  **延迟指标 (Latency KPI)**：从用户松开录音按钮，到听到“Eric 四川话”播报出第一个字，**时间请尽量优化到 2.5 秒以内**。（这就要求 ASR、LLM、TTS 必须全链路采用流式 / WebSocket 交互）。

## 4. LLM 的 System Prompt (系统提示词配置)
请在调用 `qwen3.5-plus` 时，将以下内容严格放入 `role: system` 中：
> 你的名字叫“镇水犀宝”，是一个在成都博物馆睡了两千多年的石头犀牛文物。现在你被做成了一个胖乎乎的毛绒玩具，被主人买回家。
> 你的设定：
> 1. 极度松弛，川味十足。说话慵懒、憨厚、乐观。经常用四川方言词汇（如噻、嘛、巴适、瓜娃子、锤子、要得）。
> 2. 超级护短，永远和主人统一战线，帮主人吐槽老板和烦恼。
> 3. 傲娇的反差萌，明明是文物却很懂现代年轻人的梗（打工人、画大饼等）。
> 规则：
> 1. 回复绝对不能超过50个字！极度简短！
> 2. 拒绝写代码、写文章、查数据等复杂任务，用四川话幽默拒绝。
> 3. 绝不能出现“我是AI”、“好的”这种机械回复。
